Repeating, using #cycle

20/2/2016

#cycle?

This week's blog will focus on the enumerate method, #class. To first understand what #cycle does, we must first have a basic comprehension of what enumeration does in Ruby.

The general definition of enumeration, or to enumerate, goes something along the lines of "...naming things separately, one by one."

Enumerators: Read up!

If we take this definition and make it relevant to this discussion, in Ruby enumeration is "calling elements, such as strings, integers, and other objects like 'nil', separately inside of a list, and performing an action one-by-one."

Getting back to specifics, #cycle targets every single element inside of an array or hash a certain amount of times.

If you pass a block after calling #cycles, this method will allow you to execute the block on every element inside of the array.

For example:
array = ['apples','oranges','bananas','grapes']
p array.cycle(2) {|fruit| puts fruit}

Should hopefully return:
=>'apples' 'oranges' 'bananas' 'grapes' 'apples' 'oranges' 'bananas' 'grapes'

Be careful though! If you do not provide an argument for this method you will return an infinite amount of elements inside the array. When I pass the parameter (2) to the enumerator #cycle, it 'puts' 2 times before it quits.

What else?

Of course this just scratches the surface to what #cycle can really do. Other methods can be chained to #cycle, and very interesting things can happen.
