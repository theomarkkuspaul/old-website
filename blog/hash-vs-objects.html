The Same But Completely Different...

I'll admit it, there is not a whole lot of difference between Objects in JavaScript and Hashes in Ruby. Maybe change a few words here, change a bit of the syntax there, a big generalisation would be that they are exactly the same thing. It is true that they act very similarly in that they both store values which can be accessed by using a label.

Let's see what we know about the two:

JavaScript Objects could be created a little something like this:

var ball = {colour : "blue", size : "large", weight : 200g}

Ruby is practically the same, except for that rascal 'var' declarative

ball = {colour : "blue", size : "large", weight : 200g}

They are very close to being the exact same syntax right?

One difference is that in Ruby each label that appears before the colon is called a 'key' whereas these are called 'properties' in JS. It is hard to pin down an exact reason why they have different names. Perhaps it has something to do with the fact that label values can be accessed differently.

To access a key's value in Ruby:

ball[:colour] => "blue"

Whereas in JS, you have two different notations: dot and bracket.

ball.colour => "blue"

ball["colour"] => "blue"

One thing to note is that you can only set a property as a string in JAva Script. In Ruby you might set a key as a string, integer, array, or even another hash, et cetera. JavaScript has a stricter convention.

The complexity of Ruby

hash = {[1,2,3,"hello"] => "hash value", {"number" : 4} => 3, "Another string to complicated things further"}

Hashes can get pretty crazy in Ruby if you don't watch yourself

His opponent: JavaScript

object = {vase : "flowers", fingers : 10}

Ok... I lied earlier, so maybe there is ONE significant difference.

Since I'm on a role another thing to note is in JavaScript you are able to set a function as a property value. In Ruby you cannot set a value of a key to a method. Take a look at this example:

hash = {key: def add(num1,num2)}

If you were to complete the rest of that method, you would run into all sorts of errors.

In JS however you are able to pull this of.

object = {add : function() {
console.log(1 + 2 + 3)
}

And this is actually a feasible way of storing data, especially if you pump in parameters. You can do a lot with this. If you need to archive the output of a calculation and call upon this output in another function, objects are very good at facilitating this.
