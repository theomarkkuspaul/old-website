Inline vs. Inline Block

6/2/16

An Overview

Using the "display" property with "inline" and "inline-block" values can be a powerful way to organise elements on a webpage. But what are the main differences and advantages between the two? When should I use one over the other?

Inline

One way to imagine this property-value is to think of an element or object that acts as text behaves. When you surround a box by text, the text and the box should flow neatly together. If an inline element is placed next to text, it should act like text and sit next to it.

Inline:

  • Will ignore the top and bottom margin values but still applies left and right.
  • Allows all padding values
  • Isn't affected by width and height properties
  • Can be aligned
  • Will be changed to a block-level element if float is applied to its element
Inline values are best used when you are trying to format text around an element. Say you are trying to add a description of an image, you would set the img's display value to inline so the text would mould around it.

Inline Block

Robert Nyman had this to say about inline blocks, "Basically, it's a way to make elements inline, but preserving their block capabilities such as setting width and height, top and bottom margins and paddings etc."

Inline Block:

  • Can place several elements that act like blocks on the same horizontal line.
  • Resize the element using width and height properties
  • Complete access to padding, borders and margins
